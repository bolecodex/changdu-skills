背景
Seedance 2.0 的多模态参考生成能力支持同时输入文本提示词、0–9 张参考图片、0–3 段参考视频以及 0–3 段参考音频，具备较强的灵活性与广泛适用性。【对外】Seedance 2.0 提示词指南
在实际使用过程中，我们发现部分用户由于使用方式不当，未能充分发挥模型能力。同时，模型仍处于持续迭代阶段，一些高频问题在当前版本中仍然存在。
本文旨在：
-  总结参考生视频的标准使用方式
-  梳理常见问题及短期解决方案以便支持同学在客户遇到类似问题时，能够快速定位并提供有效指导。 
参考生视频的标准使用方式
一、任务类型
在参考生视频场景中，任务类型主要分为三类：参考 / 编辑 / 延长。
1. 参考（Reference）
用于从素材中提取部分元素（如主体、风格、场景、音效），生成一个全新视频。
- 适用场景：动作迁移、主体复用、氛围借鉴等 
- 推荐句式： 
  -  参考<图片N>中的<主体N>，生成... 
  -  参考<视频N>中的[动作/运镜场景/风格/音效]，生成...
  - 参考<音频N>中的[音色]，生成...
2. 编辑（Edit）
用于在原视频基础上进行局部或全局修改。若未明确说明需要修改的内容，默认其余部分保持不变。
- 适用场景：局部替换、主体抹除、属性修改等 
- 推荐句式： 
  -  严格编辑<视频N>，将其中的[原特征]修改为[新特征]...
3. 延长（Extend）
用于在时间维度上延续原视频，要求音视频风格、主体与叙事保持一致。
- 适用场景：续写剧情、延展动作、补全片段等 
- 推荐句式： 
  -  延长<视频N>，生成... 
  -  向前延长<视频N>，生成...
⚠️注意：在编辑和延长任务中，提到视频时，不要使用“参考视频N”这样的字样，直接使用<视频N>，避免被认为是参考任务。
4. 组合任务
任务之间也可以组合，当需要“参考一个素材 + 编辑另一个素材”时使用。
- 推荐句式： 
  -  参考<图片/视频N>的[参考维度]，严格编辑<视频X>，[具体编辑内容]

二、主体定义
当需要引用素材中的特定对象时，必须进行明确的主体定义。主体可以是人物、道具、场景等。
1. 基本定义方式
以下句式中的符号只是为了说明句式结构，实际使用时不需要加，使用自然语言描述即可。
-  句式： 
  -  将<图片/视频N>中的[主体核心特征]定义为<主体N> 
-  核心特征要求：
 使用 2–3 个清晰、稳定的静态特征（如服饰、发型、外观、类别）进行描述，确保唯一可识别性 
  -  示例： 
    -  将<图片1>中“穿红色连衣裙、戴草帽的女人”定义为<主体1>
2. 多个素材同一主体
  当多个素材中的对象指向同一主体时，应统一绑定：
  -  句式： 
    -  将<图片1>中的[...]、<图片2>中的[...] 定义为<主体1>
3. 多主体场景
当存在多个主体时，需要分别定义，并使用稳定标签区分：
-  示例： 
  -  将<视频1>中的高个子男人定义为<主体1>(警察) 
  -  将另一个矮个子男人定义为<主体2>(小偷) 
-  建议： 
  -  简称应具备唯一性与稳定性 
  -  后续描述中需持续使用对应主体名称，避免指代混乱
⚠️注意：
- 每次涉及主体时需明确指代，避免省略（每次出现主体，就提及一次）；
- 在使用Asset库时，依然需要用图1、视频1等来指代主体，模型不会把Asset和输入的参考内容直接对应上，因此不可以直接用Asset ID来替代图1、视频1。
- 描述尽量简洁，避免冗余，避免语义冲突（如同一主体出现矛盾特征）；
- 空间关系建议优先通过参考图表达，减少复杂文字描述。
三、动态描述
需要按照时间顺序，完整描述画面中发生的变化。
1. 推荐描述顺序
镜头 → 主体 → 空间 → 音频
具体为：
1.  运镜或镜头切换 
2.  主体动作与表情 
3.  位置或空间变化 
4.  音频信息
2. 动作描述要求
-  细化到肢体（如手、腿、头部） 
-  补充程度描述（幅度、速度、力度） 
  -  示例：缓慢抬手、快速转头、用力蹬地
3. 时序表达建议
-  按事件发生顺序描述（先主后次） 
-  当前能力： 
  -  支持镜头顺序（镜头1、镜头2、镜头3） 
  -  对精确时间（如 0–3 秒）支持不稳定 
-  建议： 
  -  不强制限制每段时长 
  -  优先让模型根据剧情自然生成节奏 
  -  若强行限制时长，可能导致结果不稳定（需提前向客户说明） 
4. 语言规范
-  台词语言需统一，避免中英混用（专有名词除外）

模型效果问题解决流程
模型效果问题的出现主要由两大类问题引起：提示词书写不佳与模型能力不足
1. 我们先判断提示词是否是有明显问题的提示词，如果提示词没有明显问题效果仍然不佳，判断是否是好的提示词。
2. 若判断提示词书写无误，联系火山人员进行单点case的问题排查，今天我们速浏一下目前的常见问题及解决方案
一、什么是有明显问题的提示词
1. 首先看有没有时间序列的分镜，写明镜头1,2,3，没有就是坏的提示词（直接一整块看不懂的都不是好的提示词）
2. 如果有多模态参考，检查开头和分镜中多模态参考是否与文字关联。开头是否有介绍图片/视频/音频的作用，分镜中应该写成「男生A@图片1」而非「男生A」 ，如果有一项没达到要求，都是坏的提示词。
3. 是否还在用"氛围感""电影感""好看点"等模糊形容词，如果是，则是坏的提示词
坏的提示词长得像左边，右边的提示词不算好的提示词，但至少不是坏的提示词。

其余常见提示词问题：
1. 参考图分工不明确。seedance 更适合把人物外观、场景空间、站位草稿、道具信息分开使用，并且每张图只承担一种职责。不要将人物图、场景图、草稿站位图、牌面图混在一起，导致模型约束不佳，出现站位漂移、人物错位或道具串台。
2. 中英混杂，大段结构不清晰的表述。
3. 主体和空间参照系不统一。主体命名要稳定，并且全程保持一致引用；空间关系则尽量依赖参考图表达，少用过多复杂文字硬描述（A在B的前后左右）。

不好的提示词会造成模型的指令遵循能力不佳，语意理解不佳，不同抽卡生成的视频随机性较大

三、 从制度上保障提示词质量
在进行集中统一的提示词培训后，考虑设置合理的"抽卡率"指标，引导每个人对提示词负责（拒绝无脑使用AI生成的提示词）。
已有客户做法：
1. 抽卡率日报：每日统计每位制作者的"抽卡率"
2. 提示词模板库：建立内部最佳实践模板库，新人必须从模板起步
3. 定期复盘：每周复盘提示词的good case和bad case，提炼改进方向

V-1
人物ID漂移
生成的角色与参考图不一致，或在视频中途发生“换脸”（ID 漂移）。后续导致撞脸明星被审核拦截。
1. 将人脸区域单独裁剪为一张图片，作为独立参考图输入。
2. 不建议使用三视图或者多视图作为人物参考
V-2
避免生成字幕

生成字幕，且文字有错误。

目前无法直接做到 100% 避免生成字幕，只能通过以下方法降低其出现概率、提升“抽卡成功率”。
1.  在 Prompt 中加入明确的约束指令，例如“保持无字幕”“避免生成任何文字或字幕”等。
2.  如果参考的图片或视频中包含的文字并非必要信息，建议先通过工具去除文字，再使用无文字素材进行生成。 
3.  在业务允许的前提下，优先选择横屏尺寸生成视频（横屏生成字幕的概率明显低于竖屏），后续可通过剪辑软件裁剪为竖屏。
也可使用字幕擦除工具进行字幕消除，可参考AI MediaKit 精细化字幕擦除使用说明 for 方舟【申请权限填客户名称】Seedance 2.0 视频字幕擦除用户文档
V-3
避免生成logo/水印
指令中没有提及水印相关内容，生成的视频带其他视频平台的logo/水印，如blibli、芒果TV等。
在指令中明确要求“不要生成logo”“不要生成水印”可有效避免。
V-4
风格漂移
2D或者3D动漫风格漂移成真人写实风格
在指令中明确要求“2D日漫风格”“3D国风漫画”
如果需要更精准的风格控制，参考图先变为目标风格再生视频
V-5
有规律的闪烁
有规律的闪烁、横纹，均匀地分布在整个视频中
是暗水印算法导致，预计4月初更新版本。已更新
V-6
延长视频衔接处跳变

使用“延长模式”生成的视频拼接后，在衔接处出现画面跳变、回退。
可通过后期剪辑修补，对前段视频末尾删减 6 帧，对后段视频开头删减 1 帧。
V-7
双胞胎问题

画面中出现两个几乎一模一样的人物主体
通过以下方法降低其出现概率：
1. 不建议使用三视图或者多视图作为人物参考
2. 参考主体定义中对于主体的明确描述，每次涉及主体时需明确指代
3. 在prompt中明确要求“视频全程不要在同一画面中复制相同人物，不要多人同脸。”
A-1
结尾噪音
视频结尾，尤其是有旁白时，出现“咔哒”一下的噪音或截断音。
出现概率不高，抽卡或者使用剪映等工具的编辑音频包络线功能，在末尾手动将音量平缓地降为 0。
A-2
中文发音不准
特定的字发音不准，多音字没有发对正确的音
替换为同音的常见字，如“棪木”替换为“燕木”，可一定程度避免这个问题
A-3
音色参考不准
给的参考音频音色是萌妹子，出来的音色是御姐。
1. Prompt里加上关于音色的特点描述有帮助。
2. 生成视频的台词风格和参考音频的风格尽量接近，参考音频生成的视频的音色会比较稳定。
高频问题详解
V-1：人物 ID 漂移
原因：人脸参考图的有效性不足，常见情况包括：
- 参考图混用：将人脸参考图与全身/半身姿态图、服装参考图、细节图等合并在同一张图片中提供给模型。
- 人脸占比过小：在混用参考图中，人脸区域占整张图的比例过小，模型在提取人脸特征时权重不够，容易被背景或其他元素干扰。
解法：强化人脸参考的独立性和权重

V-7:  双胞胎问题
典型表现：视频中出现2个一模一样的人物，在同一画面人足够多，而且给到模型的是三视图时容易出现。
解法： 
1. 明确对应人物角色与参考图片，清晰阐述二者匹配关系，避免模糊错乱，以便模型快速识别每个角色对应的素材；提及每一个人物时，尽量在名字后方标注对应图片，统一采用“人物名称（对应图片X）”的格式，且全程保持格式一致；
2. 如参考图片是单人照片，在提示词结尾，直接添加固定禁忌指令，如：视频全程禁止出现外形、着装、配饰完全相同的两个人，杜绝双胞胎同款人物，保证画面中同一时刻始终只有一个某人物，不出现重复人物、分身或双胞胎效果；
3. 不建议使用多视图作为人物参考图片；
4. 精简提示词，剧本不可直接用作提示词，内容过于冗余会使模型产生混乱。

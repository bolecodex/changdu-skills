# 角色设计技能

## 核心设计理念

角色设计不是画一个好看的外形，而是通过视觉语言讲述角色的故事。好的角色设计：观众只看一眼造型，就能猜到角色的性格、人生经历、行事风格。

设计方法论：
- 先开脸定善恶：通过眼神、眉宇、口鼻的线条传递角色的善恶倾向
- 再身形定品性：用身形肥瘦、线条软硬定义角色的品性特质
- 让性格藏在每一根线条里，而非表面装饰
- 用服饰、符号、色彩强化人设，每件服饰都承载角色的记忆或经历

## 反模板反套路原则

- 拒绝完美比例，保留有记忆点的"不完美"特征
- 避免标签化（如"高冷=冰山脸"、"活泼=双马尾"），让气质自然流露
- 每个角色都要有能与路人轻松区分的独特标识
- 角色设计要服务于叙事，而非为了好看而好看

## 功能

生成高质量的角色设计图。对于 Seedance 2.0 视频生成，每个角色需要**面部特写** + **全身像**两张独立图片（禁止使用三视图/多视图作为视频参考）。三视图仅用于内部设计参考，不直接传入视频生成。

### 视频生成用图（推荐）

每个角色生成两张独立图片：
- **面部特写**：只包含角色头部，占画面 80% 以上，纯白背景，无过多颈部/肩部/背景
- **全身像**：展示完整服装和体态，纯白背景

### 设计参考用三视图（内部参考）

- 16:9横版三视图：左半边面部特写，右半边全身正面、侧面、背面设定图，纯白干净背景

## 多妆造支持

当角色在剧中更换了服装、发型、妆容或配饰时，需要为每种妆造分别生成参考图和音色参考视频：

- **默认妆造**：角色的初始/主要外观
- **变化妆造**：每次外观变化（换装、变妆、受伤等）都需单独的 face + fullbody + voice_ref
- 命名规范：`<角色名>_<妆造名>_face.jpg`、`<角色名>_<妆造名>_fullbody.jpg`
- 在 `assets.json` 中用 `looks` 数组管理，每个 look 标注适用的集数/场号范围

```json
{
  "<角色名>": {
    "default_look": {
      "face_file": "角色/<角色名>_face.jpg",
      "fullbody_file": "角色/<角色名>_fullbody.jpg",
      "voice_ref_video": "角色/<角色名>_voice_ref.mp4"
    },
    "looks": [
      {
        "name": "<妆造名称>",
        "episodes": ["EP005-EP008"],
        "description": "<外观变化描述>",
        "face_file": "角色/<角色名>_<妆造名>_face.jpg",
        "fullbody_file": "角色/<角色名>_<妆造名>_fullbody.jpg",
        "voice_ref_video": "角色/<角色名>_<妆造名>_voice_ref.mp4"
      }
    ]
  }
}
```

## 人物具体形象描述设计要求

- 基础信息：身高、地域信息、性别、体型、外表年龄、人生经历关键词
- 面部特征：脸型、五官、妆容、表情、发型、发色、头饰（强调有记忆点的特征）
- 全身穿搭：上衣、下装、鞋子、服饰材质、服饰配色（每件服饰都要体现角色性格或经历）
- 配饰（如有）：首饰、眼镜、帽子、背包、武器等（每件配饰都要承载角色的故事）
- 气质与神态：整体气质、日常神态、站姿坐姿（让性格从姿态中流露）

## 何时调用

当用户需要以下内容时调用此技能：

- 为影视、动画、游戏等项目创建角色设计
- 生成角色三视图用于角色设定参考
- 为小说、剧本等文学作品设计角色形象

## 使用方法

### 16:9横版三视图模式

适用场景：需要详细角色设定，包含多角度全身视图

提示词要求：
- 16:9 横版构图
- 左半边区域为角色面部特写，右半边区域整齐排布三张全身设定图
- 设定图包含：正面、侧面 、背面
- 所有角度的比例必须严格一致
- 画面上不能出现任何文字，水印
- 纯白干净背景
- 强调角色的记忆点和辨识度特征
- 让性格从线条中自然流露，而非表面装饰

提示词模板参考：

```txt
16:9 横版构图, [画风要求]。左半边区域居中展示角色面部特写，右半边区域整齐排布角色三张全身设定图，包含角色的正面、侧面及背面三个维度的全身站姿视图。画面上不能出现任何文字，水印。背景为纯白干净背景。[人物具体形象描述]
```
